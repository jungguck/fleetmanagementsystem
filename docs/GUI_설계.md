## 2. ⚠ 가장 중요한 설계 제약 — 폼은 자동갱신 밖에 둔다

대시보드는 매 0.5초 `dash.refresh()` 로 **통째로 다시 그린다**.
입력 위젯(`ui.select`)을 그 안에 두면 **매 0.5초 리셋돼서 운영자가 고를 수가 없다.**

```python
# main.py
app.timer(state.poll_interval, state.refresh)   # ← 관제 두뇌: 앱 전체에 '하나'

@ui.page("/")
def index():
    control.control_panel(state)   # ← 한 번만 렌더 (select 가 있으므로 refresh 밖)
    tasks.create_form(state)       # ← 한 번만 렌더 (동일)

    @ui.refreshable
    def dash() -> None:
        dashboard.dashboard_body(state)   # ← 읽기 전용이라 매 틱 재생성 OK
    dash()

    ui.timer(state.poll_interval, dash.refresh)  # ← 이 브라우저의 '화면만'
```

**⚠ 관제 로직을 `ui.timer` 로 돌리면 안 된다.** 페이지 함수는 **브라우저마다** 실행되므로
`ui.timer(…, state.refresh)` 는 접속자 수만큼 생긴다:
- 브라우저 **0개** → `refresh` 가 아예 안 돎 → **작업이 배차되지 않는다**(관제 정지)
- 브라우저 **2개** → 배차·배터리 계산이 **2배속**

→ 관제(`state.refresh`)는 `app.timer`(앱 1개), 그리기(`dash.refresh`)는 `ui.timer`(페이지별).

| 위젯 | 위치 | 이유 |
|---|---|---|
| `control_panel` (목적지 select·teleop·E-STOP) | refresh **밖** | select 값 보존 |
| `tasks.create_form` (픽업/드롭 select) | refresh **밖** | select 값 보존 |
| `dashboard_body` (맵뷰·카드·작업큐) | refresh **안** | 읽기 전용 |

## 3. 맵뷰 — "웹에 그린 turtlesim 창" **[구현됨]**

`gui/ui/mapview.py`, SVG 520×520.

**turtlesim 창과 같게 그리는 것**
- 배경: turtlesim 기본 파랑 `#4556ff` (`models.TURTLESIM_BG`).
- 거북이: `/turtleN/pose` 실측 위치. 등껍질·머리·발 4개를 `r.pen` 색으로.
- 펜 궤적: 지나온 자취(실선). turtlesim 의 `set_pen` 과 **같은 `r.pen` 색**을 쓴다.

**turtlesim 창이 모르는 것 (관제 오버레이)**
- **A\* 계획경로**: 남은 웨이포인트를 상태색 **점선** + 목표점 링. ← 관제의 핵심 부가가치
- 스테이션 `▣`(pickup 주황 / drop 보라 / charge 초록) + 이름(마커 아래).
- 가상벽: 반투명 흰 사각 + 점선 테두리(실물이 아니라 관제 개념이라 흐리게).
- 상태색 링: 거북이를 둘러싼 링 색 = `STATE_COLORS[state]`.
- OFFLINE: `opacity 0.35` 로 흐려짐.

**좌표 변환** (`to_screen`)
```
px = (x - xmin) / (xmax - xmin) * W
py = H - (y - ymin) / (ymax - ymin) * H     # 화면 y 는 아래로 +, 월드는 위로 + → 뒤집기
```
`world` 는 `config.yaml` 단일 정의처에서 맵뷰·플래너가 함께 읽는다(`state.world`).
거북이 회전은 `rotate(-degrees(yaw))` — y 를 뒤집었으니 각도 부호도 뒤집는다.

**색 배정 규칙 (중요)**
- `PEN_COLORS`(로봇 = 노랑/분홍/청록/주황)는 **`STATE_COLORS` 와 절대 겹치지 않게** 고른다.
  겹치면 화면의 색이 "로봇을 뜻하는지 상태를 뜻하는지" 알 수 없다.
- 둘 다 파란 배경 위에서 읽혀야 하므로 밝은 색 위주.

**미구현**: 맵 클릭 → 목표 전송, 로봇 선택, zone 점유 표시.

**왜 SVG 인가**: 도형 수백 개뿐이라 매 틱 통째로 재생성하는 게 부분 업데이트보다 단순·충분.

## 4. 로봇 상태카드 **[구현됨]**

`gui/ui/dashboard.py: robot_card()` — 카드 1장 = 로봇 1대.

- 헤더: 이름 + **상태 표시등**(색 점 = `STATE_COLORS[state]`).
- `상태: driving` (상태색 글자) / `작업: PICK-A→DROP-B (적재)`.
- 배터리 바: `LOW_BATT(20%)` 미만이면 빨강 → 자동 충전복귀 대상이라는 신호.
- `🔋 78%  📍(4.2, 1.8)` — 위치는 `/turtleN/pose` 실측.
- `online == False` 면 `⚠ OFFLINE — /pose 없음`.

**미구현**: 카드별 `[목표전송][정지][teleop][상세]` 버튼(조작은 상단 패널에 모아둠),
경로 진행률 바, fault 배지.

## 5. 조작 패널 **[구현됨]**

`gui/ui/control.py` — 조작 방식이 **2가지**이고, 이 둘의 차이가 이 프로젝트의 핵심 개념이다.

| 방식 | 누가 고르나 | 동작 |
|---|---|---|
| **목적지 지정** `[스테이션▾][이동]` | 운영자가 **로봇도 목적지도** | `send_to_station()` → A* → 자동주행 |
| **수동 teleop** `▲◀▶▼■` | 운영자가 직접 운전 | `teleop()` → `cmd_vel` 직발행 |
| **자동 배차** (§6 폼) | **시스템이 로봇을 고름** | 작업만 만들면 최근접 로봇이 수행 |

**teleop 상세**
- `▲`=(전진 2.0, 0) `◀`=(0, +1.5) `▶`=(0, -1.5) `▼`=(-2.0, 0) `■`=정지.
- 누르면 **`TELEOP_HOLD`(1초) 동안만** 움직이고 자동 정지 → 계속 가려면 연타.
  (버튼은 "누르고 있는" 이벤트가 없으니 시간 제한으로 대신한다. 폭주 방지도 겸함.)
- **수동이 자동을 override 한다**: `teleop()` 이 `path` 를 비우고 `state="manual"` 로.
- 만료되면 제어루프가 `manual=None, state=idle` 로 복귀시킨다.

**루프 클로저 함정** — 버튼을 for 문으로 만들 때 반드시 기본인자로 캡처:
```python
ui.button("이동", on_click=lambda rid=r.id, d=dest: state.send_to_station(rid, d.value))
#                              ^^^^^^^^^^^^^^^^^ 이게 없으면 전부 마지막 로봇을 가리킨다
```

**E-STOP / 해제**: 전 로봇 정지 + 작업 취소 + **래치**(시나리오8).
래치 중엔 [이동]·teleop 이 거부되고 알림이 뜬다(조용히 무시하면 고장으로 보인다).
`[🔓 해제(RESET)]` 는 **래치가 걸렸을 때만** 보인다:
```python
ui.button("🔓 해제(RESET)", ...).bind_visibility_from(state, "estopped")
```
이 패널은 refresh 밖(한 번만 렌더)이라 매 틱 다시 그려지지 않는다 → 조건부 표시는
`if` 문이 아니라 **바인딩**으로 해야 한다(§2).

**궤적 지우기**: `clear_trails()` — turtlesim 의 `/clear` 에 해당하는 화면 청소.

## 6. 작업 패널 **[일부]**

`gui/ui/tasks.py`
- **작업 생성 폼**(refresh 밖): `[픽업▾] → [드롭▾] [추가]`. 픽업=드롭이면 경고.
- **작업 큐**(refresh 안, 읽기 전용): `대기 N · 수행 N · 완료 N` + 최근 8건.
  행: `#id  PICK-A → DROP-B  running  [turtle3]`. 상태색: pending 회색 / running 파랑 / done 초록 / cancelled 빨강.

**미구현**: 우선순위, 행별 `[취소][우선]`, 진행률 바.

## 7. 색 / 상태 규칙 (시나리오.md 와 공유 — 정의처는 `models.py`)

| State | 코드 | 카드 표시등 | 맵 링 |
|---|---|---|---|
| `idle` 대기 | `#9aa7b0` 회색 | ⬜ | 회색 링 |
| `driving` 주행중 | `#2e86de` 파랑 | 🟦 | 파랑 링 + 점선 경로 |
| `waiting` 양보대기 | `#e67e22` 주황 | 🟧 | 주황 링 + 안전거리 원 |
| `manual` 수동조작 | `#16a085` 청록 | 🟩 | 청록 링 |
| `charging` 충전중 | `#27ae60` 초록 | 🟩 | 초록 링 |
| `error` 에러 | `#e74c3c` 빨강 | 🟥 | 빨강 링 |

한글 라벨은 `models.STATE_LABELS` 단일 정의처에서 가져온다(카드에 `driving` 대신 `주행중`).

**교통관제 표시** (`traffic.py` 와 연동): 위험 쌍을 **빨간 점선**으로 잇고, 양보 중인 로봇 둘레에
**주황 안전거리 원**을 그린다. → "관제가 지금 무엇을 보고 세웠는지"가 화면에 보인다
(안 보이면 운영자는 그냥 멈춘 걸로 오해한다).

- 배터리: `< LOW_BATT(20%)` 빨강 / 그 외 초록.
- OFFLINE: 카드에 경고 문구 + 맵에서 반투명.
- **단일 정의처**: `models.STATE_COLORS` / `models.PEN_COLORS` 를 맵뷰·카드가 함께 import.
  (색을 UI 파일마다 하드코딩하면 화면끼리 어긋난다.)

## 8. 데이터 흐름

```
[NiceGUI 화면] ──클릭/폼──▶ FleetState (send_to_station / teleop / add_task / stop_all)
      ▲                              │ path 갱신 (FLEET_LOCK)
      │                              ▼
      │                   turtlesim_source 20Hz 제어루프 ──/turtleN/cmd_vel──▶ turtlesim
      │                              ▲
      └── @ui.refreshable ◀── RobotState ◀── _on_pose 콜백 ◀──/turtleN/pose──┘
           (0.5s ui.timer)
```

**주기가 2개인 게 핵심**:
| 주기 | 누가 | 무엇 |
|---|---|---|
| **20Hz** (ROS 실행기 스레드) | `turtlesim_source._control()` | cmd_vel 발행·도착판정·배터리 |
| **0.5s** (NiceGUI 스레드) | `ui.timer` → `refresh()` + `dash.refresh()` | 배차·작업FSM·화면 그리기 |

제어를 0.5s 로 하면 거북이가 뚝뚝 끊기고, 화면을 20Hz 로 그리면 브라우저가 죽는다 → 분리.
대신 **두 스레드가 같은 `RobotState` 를 만지므로 `models.FLEET_LOCK`(RLock)으로 보호**한다.
