# FMS 구현 로드맵 / 태스크

> `README.md` 의 로드맵을 **자기완결적 소태스크**로 쪼갠 문서.
> 남은 태스크는 그대로 **구현 프롬프트**로 붙여 쓸 수 있게 `[Pn-k] 목적 / 할 일 / 산출물 / 완료기준` 형식.
>
> **P0~P3, P5 는 대부분 완료됨.** 아래 §완료 를 먼저 보고, 남은 건 §남은 작업 에 있다.

**공통 규약**
- ROS 2 **Jazzy 네이티브**, 시뮬은 **turtlesim** (TurtleBot3/Gazebo/nav2/Docker 안 씀).
- 로봇 네임스페이스 `/turtle1 /turtle2 …` (= `config.yaml` 의 `id`).
- 실행은 **python3.12 venv** 필수 (`docs/환경.md` §1 — rclpy 가 3.12 빌드).
- 각 태스크 완료기준은 **라이브 실행 증거**(실제 거북이가 움직인 로그/좌표)로만 통과.
  단위테스트 통과나 "코드 짰음"은 완료가 아니다.

## ✅ 완료 (실측 검증됨)

| 단계 | 내용 | 완료기준 | 코드 |
|---|---|---|---|
| **P0 환경** | Jazzy + turtlesim + python3.12 venv | 거북이 4마리 spawn | `run.sh`, `docs/환경.md` |
| **P1 단일** | pose 구독 + cmd_vel 제어 + 웹 목표전송 | **웹 [이동] → 거북이 도착** | `turtlesim_source.py` |
| **P2 다중** | 4마리 동시 제어 + 다중카드 + 맵뷰 N pose | 4대 동시 주행 | `config.yaml`, `dashboard.py` |
| **P3 작업/배차** | 작업큐 + 최근접 배차 + 작업 FSM | 작업 넣으면 자동 선택·수행 | `state.py: dispatch/_advance_tasks` |
| **P3+ 경로계획** | 격자 A* (nav2 대체) | 벽 피해 돌아감 | `planner.py` |
| **P5 충전** | 가상 배터리 + 저전력 자동 충전복귀 | 저전력 → 충전 → 복귀 | `state.py: _auto_charge` |
| **P5 안전** | E-STOP 래치 + 해제(RESET) | 클릭 → 즉시 정지, 스스로 재출발 안 함 | `state.py: stop_all/reset` |
| **P3 고장대응** | 통신두절 → error + **작업 자동 재배차** | 거북이 kill → 작업이 다른 로봇에 승계 | `state.py: _requeue` |
| **P5 수동** | teleop (cmd_vel 직발행, 1초 홀드) | ▲ → 거북이 전진 후 정지 | `control.py`, `state.py: teleop` |
| **GUI** | 웹에 그린 turtlesim 화면 + 관제 오버레이 | 브라우저만으로 관제 | `mapview.py` |
| **P4 교통관제** | 충돌 감지 + 우선순위 양보(교착 없음) | 교차 시 한 대가 서고 지나간 뒤 재개 | `traffic.py` |

**P0~P2 가 통째로 사라진 이유**: SLAM 맵작성 / AMCL 로컬라이제이션 / Gazebo world /
nav2 파라미터 튜닝 / 네임스페이스별 nav2 스택 / tf prefix — turtlesim 은 `/turtleN/pose` 로
**오차 없는 위치를 그냥 준다.** 이 문제들이 존재하지 않는다.
그 대가로 잃은 것: 센서(LiDAR)·동적 장애물·물리·실기 갭. 관제 로직 학습엔 지장 없다.

## 🔜 남은 작업

### [P4-2] 정면대향 우회 재계획 (P4-1 의 남은 절반)

- **왜**: 지금은 양보하는 로봇이 **선 자리가 곧 상대의 경로**라서, 정면으로 마주치면
  우선순위 로봇이 그냥 통과해 버린다(실측 최소 접근거리 0.15 — 겹침).
  교차 상황은 잘 되지만 1폭 통로 대향은 못 푼다.
- **할 일** (택1):
  - (a) **임시 장애물 재계획**: `GridPlanner.plan(start, goal, blocked_pts=[(x,y,r), …])` 를 추가해
    '서 있는 로봇'을 막힌 셀로 넣고 우선순위 로봇이 우회 경로를 다시 뽑게 한다.
    우회로가 없으면(1폭 통로) 저순위 로봇을 **후퇴**시켜야 하므로 (b) 가 필요.
  - (b) **일방통행 규칙**: config 에 통로별 방향을 주고 A* 비용에 반영.
- **주의**: 재계획을 20Hz 루프 안에서 하면 안 된다(A* 는 락 밖·저빈도로 — README 3원칙).
- **완료기준**: turtle1(2,2)→(9,2), turtle2(9,2)→(2,2) 로 동시에 보냈을 때
  **최소 접근거리가 safe_dist 아래로 안 떨어진다**(현재 0.15 → 목표 >1.2).
  검증: `scratchpad/e2e3.py` 의 min_dist 측정을 assert 로 승격.

### [P3-6] 작업 우선순위 / 개별 취소 (시나리오6)

- **목적**: 지금은 FIFO 고정이고, 취소는 E-STOP 전체취소뿐.
- **할 일**: `Task.priority` 추가 → `dispatch()` 에서 pending 우선순위 정렬.
  작업큐 행에 `[취소]` 버튼(단, 폼이 아니라 읽기전용 영역이라 **refresh 밖으로 옮기거나**
  `ui.refreshable` 구조를 바꿔야 함 — `GUI_설계.md` §2 제약 주의).
- **완료기준**: HIGH 작업이 먼저 배차되고, 개별 취소 시 그 로봇만 정지.

### [P1-6] 맵 클릭 → 목표 전송

- **목적**: 지금은 스테이션 선택만 가능. 임의 좌표로 못 보낸다.
- **할 일**: `ui.html` SVG → `ui.interactive_image` 또는 SVG `@click` 로 교체,
  클릭 px → 월드 좌표 역변환(`to_screen` 의 역함수) → `send_goal()`.
  로봇 선택 UI(카드 클릭 → 선택) 필요.
- **주의**: 맵뷰는 refresh 영역 안이라 **매 0.5초 재생성**된다 → 선택 상태를 위젯이 아니라
  `FleetState`/모듈 변수에 둬야 유지된다(`GUI_설계.md` §2).
- **완료기준**: 맵 빈 곳 클릭 → 선택된 거북이가 그 지점 도착.

### [P4-3] 알람 / 이벤트 로그

- **목적**: "언제 무슨 일이 있었나"가 지금 전혀 안 남는다.
- **할 일**: `FleetState.events: list[Event]`(시각·로봇·종류·상세, 상한 200) +
  발생 지점에서 append(배차/도착/충전복귀/OFFLINE/E-STOP) + 하단 로그 패널(읽기전용).
- **완료기준**: 충전복귀·E-STOP·OFFLINE 이 로그에 남고 화면에 보인다.

## 부록. 현재 파일 트리

```
fleetmanagementsystem/
├─ run.sh              # ROS source + python3.12 venv + 실행
├─ gui/
│  ├─ config.yaml      # 로봇·스테이션·벽·월드·백엔드  ← 대수 조정은 여기서만
│  ├─ models.py        # RobotState·Task·색·FLEET_LOCK  (최하위 = 순환import 방지)
│  ├─ turtlesim_source.py  # ROS 백엔드: spawn / pose / 20Hz cmd_vel 제어
│  ├─ state.py         # 관제 두뇌: 배차·작업FSM·충전·E-STOP + sim2d 폴백
│  ├─ planner.py       # 격자 A* (nav2 경로계획 대체)
│  ├─ main.py          # NiceGUI 페이지 + ui.timer
│  └─ ui/
│     ├─ mapview.py    # 웹에 그린 turtlesim + 관제 오버레이
│     ├─ dashboard.py  # 요약 + 로봇카드
│     ├─ control.py    # 목적지·teleop·E-STOP
│     └─ tasks.py      # 작업 생성폼 + 큐
└─ docs/               # 환경 / 시나리오 / GUI_설계 / 구현_프롬프트(이 문서)
```
