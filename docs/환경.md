# 런타임 / 개발 환경 정의 (FMS TurtleBot Fleet)

> 마스터, 이 문서는 `README.md`(타깃 ROS 2 **Humble**, 개발머신 **Jazzy/24.04** → **Docker Humble**)를 실행 환경 수준으로 구체화합니다.
> **확실치 않은 버전·패키지 조합은 "확인 필요"로 표기**(창작 금지)했습니다. 아래 "실측"은 이 머신에서 확인한 사실입니다.

---

## 0. 이 머신 실측 (2026-07, 확인됨)

| 항목 | 값 | 근거 |
|---|---|---|
| OS | **Ubuntu 24.04.4 LTS** | `lsb_release` |
| 설치된 ROS | **Jazzy** (`/opt/ros/jazzy`, `ROS_DISTRO=jazzy`) | 실측 |
| **Humble 설치** | **없음** | 실측 (24.04엔 Humble 바이너리 미지원) |
| **Docker** | **미설치** | `docker --version` 실패 → **P0에서 설치 필요** |
| GPU | **RTX 5090**, 드라이버 580.159.03 | `nvidia-smi` |
| Gazebo | `gz`(gz_tools, Jazzy 동봉)만 존재 / 독립 gazebo 없음 | 실측 |

**핵심 미스매치 (README §7-8)**: 타깃은 Humble이나 이 머신은 Jazzy/24.04. Humble은 Ubuntu 22.04 기반 → **네이티브 설치 불가**. 해결: **Docker Humble**(권장) 또는 별도 22.04 머신. Docker는 아직 미설치이므로 **P0-1의 첫 작업이 Docker 설치**입니다.

## 1. 대상 환경 (Docker Humble)

- **베이스 이미지**: `ros:humble-desktop` 또는 `osrf/ros:humble-desktop-full` — **정확한 태그·GPU 조합 확인 필요**.
- **컨테이너 안**: Ubuntu 22.04 + ROS 2 Humble.
- **호스트**: Ubuntu 24.04 + Docker + NVIDIA Container Toolkit(RTX5090 패스스루).
- **GUI/시뮬 표시**: 호스트 DISPLAY(X11) 공유 또는 (Wayland면 XWayland, 확인 필요).

```
┌ 호스트 (Ubuntu 24.04, Jazzy, RTX5090) ────────────────┐
│  Docker + NVIDIA Container Toolkit                     │
│   ┌ 컨테이너 (Ubuntu 22.04 / ROS 2 Humble) ─────────┐  │
│   │  turtlebot3 · nav2 · slam_toolbox · Gazebo      │  │
│   │  fms_backend(NiceGUI+rclpy 브릿지)              │  │
│   └────────────────────────────────────────────────┘  │
│  DISPLAY(X11) · /dev/dri(GPU) · 네트워크(host or bridge)│
└────────────────────────────────────────────────────────┘
```

## 2. 필요 패키지 (컨테이너 내부)

| 패키지 | 용도 | 비고 |
|---|---|---|
| `turtlebot3`, `turtlebot3_msgs` | 로봇 모델·메시지 | `TURTLEBOT3_MODEL`(burger/waffle/waffle_pi) |
| `turtlebot3_simulations` | Gazebo world·bringup | Humble용 브랜치 확인 필요 |
| `navigation2`, `nav2_bringup` | 주행·경로계획·회피 | 핵심 |
| `slam_toolbox` | SLAM 맵 작성 | P1-1 |
| **Gazebo 연동** | 시뮬 | **조합 확인 필요** (아래 §2.1) |
| `robot_state_publisher` | tf/URDF | 통상 bringup 의존 |
| **ROS↔웹 브릿지** | 관제 연결 | **신규**: `rclpy` 직접 임베드(권장) 또는 `rosbridge_server`(확인 필요) |
| `rviz2` | 개발 시각화 | 디버그용 |

### 2.1 Gazebo 조합 (확인 필요 — 중요)
- Humble 시절 표준은 **Gazebo Classic(11) + `gazebo_ros`**. 신형은 **Gazebo(구 Ignition) + `ros_gz_sim`**.
- TurtleBot3 Humble 데모가 어느 쪽을 기본으로 하는지 **버전·브랜치별로 다름 → 실제 설치 시 확인 필요**.
- 이 머신의 `gz`는 Jazzy 동봉이라 **컨테이너(Humble) 내부 Gazebo와 무관**(혼동 금지).
- **결정 규칙**: TurtleBot3 공식 Humble 문서가 지정하는 조합을 따른다(문서 확인 전 단정 금지).

## 3. 다중 로봇 실행 (네임스페이스)

- 로봇별 네임스페이스 `/tb1 /tb2 …`, tf 프레임 prefix, topic/액션 remap (README §2).
- **DDS 도메인**: 단일 `ROS_DOMAIN_ID` 내 네임스페이스 분리 vs 컨테이너별 도메인 분리 — **정책 확인 필요**. (일반론: 같은 fleet은 같은 도메인 + 네임스페이스 권장이나 이 프로젝트에 단정 금지.)
- **RMW**: 기본 `rmw_fastrtps` 가정, 대수 늘면 `rmw_cyclonedds` 검토(확인 필요).

### compose 예 (구조만, 값 확인 필요)
```yaml
# docker/compose.multi.yaml (초안 — 이미지/명령 확인 필요)
services:
  tb1: { image: fms:humble, environment: [ROBOT_NS=tb1], ... }
  tb2: { image: fms:humble, environment: [ROBOT_NS=tb2], ... }
  fleet_backend: { image: fms:humble, command: run_backend, ports: ["8080:8080"] }
# X11/GPU/네트워크 옵션은 §4 참조
```

## 4. 개발 vs 배포

| 구분 | 개발(시뮬) | 배포(실기) |
|---|---|---|
| 로봇 | Gazebo TurtleBot3 N대 | 실 TurtleBot3 N대 |
| 실행 위치 | 이 워크스테이션(Docker Humble) | 로봇 온보드(Pi 등) + 관제 PC |
| 센서 | 시뮬 LiDAR/odom | 실 LiDAR/IMU(노이즈·지연, README §7-9) |
| 표시 | 호스트 DISPLAY(RTX5090) | 관제 PC 웹 브라우저 |
| 리스크 | 낮음 | 로컬라이제이션 드리프트·통신·안전(실기 게이트) |

- **시뮬↔실기 갭(README §7-9)**: 센서 노이즈·지연·마찰 → 파라미터 재튜닝 필요(확인 필요). P1~P5는 우선 시뮬로 검증(README §9).
- **실기 전환 시**: `fable-work-doctrine` §5(실기 하드웨어 게이트) 준수 — 라이브 실행 증거 없이 완료 선언 금지.

## 5. GPU / DISPLAY (실측 기반)

- RTX5090 + 드라이버 580.159.03(확인됨). Gazebo/RViz 렌더링에 사용.
- Docker에서 GPU: **NVIDIA Container Toolkit** 필요(미설치 → 설치 필요). 실행 시 `--gpus all` 또는 compose `deploy.resources`(확인 필요).
- X11: `-e DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix`(+`xhost` 허용). Wayland 세션이면 XWayland 경유(확인 필요).
- **RTX5090 신형 GPU + 컨테이너 드라이버 호환**은 실제 실행으로 확인 필요(창작 금지).

## 6. Humble ↔ Jazzy 미스매치 대응 (정리)

1. **호스트에 Humble 설치 시도 금지** — 24.04 미지원.
2. **모든 ROS 작업은 컨테이너(Humble) 안에서** — 호스트 Jazzy와 혼용 금지(DDS 도메인·RMW 섞이면 디버그 지옥).
3. 호스트 Jazzy 툴(`gz` 등)을 컨테이너 대상으로 쓰지 말 것.
4. 대안(비권장): 별도 22.04 머신 네이티브 Humble(README §8). 이 프로젝트는 Docker 우선.
