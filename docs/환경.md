## 1. ⚠ python 버전 함정 (이 프로젝트 최대 삽질 포인트)

시스템 기본 python 은 **3.13** 인데, Jazzy 의 `rclpy` 는 **3.12 용으로만** 빌드돼 있다.
3.13 으로 import 하면:

```
The C extension '/opt/ros/jazzy/lib/python3.12/site-packages/_rclpy_pybind11.cpython-313-x86_64-linux-gnu.so'
isn't present on the system.
```

**해결**: python3.12 로 venv 를 만들고 ROS 를 source 한 상태로 실행한다.

```bash
source /opt/ros/jazzy/setup.bash
python3.12 -m venv --system-site-packages .venv   # --system-site-packages 필수
.venv/bin/pip install -r gui/requirements.txt
.venv/bin/python -m gui.main
```

- `--system-site-packages` : ROS 파이썬 패키지에 접근하기 위함.
- ROS `setup.bash` 를 source 해야 `rclpy`·`turtlesim` 인터페이스가 `PYTHONPATH` 에 올라온다.
- 위 과정을 `run.sh` 가 전부 자동으로 한다 → **평소엔 `./run.sh` 만 쓰면 된다.**

## 2. 필요 패키지

| 패키지 | 용도 | 설치 |
|---|---|---|
| `ros-jazzy-turtlesim` | 시뮬레이터(거북이) | apt (보통 desktop 설치에 포함) |
| `ros-jazzy-rclpy` | ROS↔파이썬 | apt (기본 포함) |
| `python3.12-venv` | venv 생성 | apt |
| `nicegui`, `pyyaml` | 관제 웹 | `gui/requirements.txt` (venv 안) |

**필요 없어진 것** (turtlesim 으로 방향 전환하며 제거):
`turtlebot3`, `turtlebot3-simulations`, `navigation2`, `nav2-bringup`, `slam-toolbox`,
`ros-gz`(Gazebo), `rosbridge-server`, Docker, `TURTLEBOT3_MODEL` env.
→ 경로계획은 `gui/planner.py`(A*), 위치는 `/turtleN/pose` 가 대신한다.

## 3. 실행

```bash
# 터미널 1
source /opt/ros/jazzy/setup.bash && ros2 run turtlesim turtlesim_node
# 터미널 2
./run.sh          # → http://localhost:8090
```

turtlesim 은 GUI 창(Qt)이라 **DISPLAY 가 필요**하다(이 머신은 `DISPLAY=:1`).
헤드리스 서버에서 돌릴 거면 Xvfb 필요 — **확인 필요**.
